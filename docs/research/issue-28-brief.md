## 问的是什么

自 #14 拆出的案头事实调查（不含 HITL 实测）：①开源模型 API 按 token 计费，海外/国内分列，折算本项目每集/每月成本；②国内托管对中国政治内容的硬阻断证据（一票否决项）；③opencode Go 套餐能否作管线后端（定价/模型/API/速率/ToS）；④Google AI Studio 免费档「用于改进服务」条款对政治逐字稿外传的含义；⑤本地 14B 约束解码（outlines/XGrammar/llama.cpp）对 20+ 字段 schema 的成熟度，为 #14 铺垫。

## 答案是什么

**成本不是问题，合规是。** 海外托管开源模型（DeepInfra/Together 的 DeepSeek-V3.x / Qwen3-235B）跑整条修复+提取，每集 ~$0.08–0.10、**每月 ~$2–6**——「免费额度不够」的成本焦虑消解。**国内托管一票否决**：《生成式AI暂行办法》第 4 条法律强制过滤，DeepSeek ~85% 拒答、Qwen 答「台湾不是国家」。**审查是两层**——API 层（仅国内端点）+ 权重内嵌（随开源权重走），海外托管只去掉第一层。**opencode Go 不建议**：$10/月给 API（OpenAI/Anthropic 兼容），但 ToS 明禁「以程序方式提取 Output」「未登录时运行的进程」，正撞 AFK 批处理，且对本项目量级比按量付费更贵。**Gemini 免费档 = 数据换免费**：官方条款明写用于训练+人工审核+「勿提交机密信息」，想关掉只能开账单（而开账单即抹掉免费额度）。**本地约束解码：schema 遵守率已解决**（XGrammar 97%+、llama.cpp GBNF 覆盖 20+ 字段子集），**幻觉率没解决**（强制必填字段出值反诱编造）。

## 对项目意味着什么

spec §6 分工表补第三选项「海外托管开源按量付费」，标注 $2–6/月、国内托管排除附审查原因；阶段 0/2 脱离 Gemini 时首选海外按量、排除 opencode Go/GLM Coding Plan；补一条 Gemini 免费档数据治理注记（须用户知情）。**交回 #14**：schema 遵守率维度可降级（工具链已成熟），HITL 火力集中到幻觉率；**新增必测项**——海外开源权重对敏感政治实体的静默漏抽（权重层审查残留）。**交回 #6**：「开账单即失去免费额度」是算账硬约束，Gemini 付费单价须与海外开源单价并排比。

## 最不可靠的地方

① opencode Go / GLM Coding Plan 的 ToS 判读——「programmatically extracts Output」在语境里偏反爬，字面却宽到覆盖合法 API 自动化；「不利于本项目」是保守判读，非黑纸白字禁令，是否可接受由用户裁。② 海外托管开源权重的「静默漏抽敏感实体」是从两层审查机制**推断**的风险，未实测——留给 #14 用含六四/习/新疆的真实逐字稿验证。③ 部分海外价来自 OpenRouter 聚合列价（非托管方一手页），Fireworks/Hyperbolic 定价页未渲染故未取数；每集成本是基于「修复 75k/75k + 提取 90k/40k」的 token 假设换算，实际用量待实跑校准（但因单价极低，结论对假设不敏感）。
