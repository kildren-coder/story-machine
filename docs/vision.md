# 下游愿景清单（vision.md）

四个下游愿景的正式落档：它们是什么、为什么现在不做、以及——本文档最有价值的部分——**每个愿景依赖基座（MVP 管线）哪几条「廉价保险」字段**。

## 来历与纪律

- 四愿景切分定于 2026-07-20 建图会话（wayfinder 地图 [#1](https://github.com/kildren-coder/story-machine/issues/1)）。
- **纪律（地图原文）**：四个下游愿景只吃基座产出。MVP 只做「字段级廉价保险」，不写任何下游逻辑。四愿景的**实现**全部在地图 Out of scope。
- 「廉价保险」的意思：有些信息**只在处理当下能拿到，事后无法补做**（比如记者名、校对配对）。现在多存一个字段几乎零成本，将来不存就永远没了。基座为每个愿景留的就是这类字段——不是功能，只是钩子。
- ⚠️ **留档警示**：建图会话本身没有留档（地图 Notes 明示）。本文系 2026-07-24 依据地图 #1 决策行、#3/#11/#14 各票结论**重建**，愿景的展开描述属合理推断而非原始记录。如与当时原意不符，以各 vision issue 里的修正为准——那里是活文档。

每个愿景对应一个 `vision` 标签的长期 issue（**不进 triage 流程**，不阻塞任何票），作为日后灵感、素材、参考项目的积累地。想到什么直接丢进对应 issue，别塞进 MVP 的活里。

## 愿景一：求真引擎

**是什么**：回头检验节目里说过的话。节目里充满断言（「历史上是这样」「现在正在发生这个」）和预测（「明年会怎样」）。求真引擎把它们变成一张可追踪的「待验证清单」：断言当期核查真伪，预测到期回头核对兑现与否，结论回填到笔记里。长期积累后，就能回答「这档节目/这位嘉宾说的话，到底靠不靠谱」。

**依赖基座的廉价保险**：

| 基座字段 | 为什么求真引擎需要它 | 证据 |
|---|---|---|
| 论断「时态×性质」二维标签 + 「存疑」兜底 | 查证队列＝直接筛 `性质:陈述`，不用回头重新分类 | #3 定稿 |
| 预测类附「期限/检验条件」字段（原文照抄 + 机械规范化时限双层） | 没有期限，预测永远「还没到时候」，无法判定兑现 | 地图决策行、#3 |
| 逐字稿（含时间戳）永久资产 + 来源记录的时间戳回链 | 争议时能回到原话原语境，避免「他真说过吗」之争 | 地图决策行、#11 |

**MVP 不做**：不建查证调度器、不做到期提醒、不自动回填结论。MVP 只保证字段在。

## 愿景二：资产分析框架

**是什么**：把节目里谈到的经济指标接上真实数据。节目常引用可追踪的数据序列（失业率、国债收益率、货币供应量……）。资产分析框架拿着提取出的「数据点」去 FRED 之类的公开数据源拉真实序列，把节目论断和真实数据摆在一起对照，支撑个人的资产判断。

**依赖基座的廉价保险**：

| 基座字段 | 为什么资产分析框架需要它 | 证据 |
|---|---|---|
| 「数据点」提取桶——收窄为「背后有可追踪序列的指标」，核心字段是**取数地址** | 有取数地址才能程序化拉真实序列；泛泛的数字没有分析价值 | 地图决策行、#3 定稿 |
| 数据点的口径原文（保真层） | 对数据前先确认节目说的口径和数据源口径是不是一回事 | #3「保真层+结构层分离」 |

**MVP 不做**：不接任何数据源 API、不画图、不做分析逻辑。MVP 只保证数据点带着取数地址躺在图谱里。

## 愿景三：故事素材库

**是什么**：攒写作素材。节目里散落着好故事——历史轶事、人物细节、精彩比喻、金句。故事素材库把它们单独标记、集中沉淀，日后写作/创作时按主题翻用，而不是靠记忆「好像哪期节目讲过一个……」。

**依赖基座的廉价保险**：

| 基座字段 | 为什么故事素材库需要它 | 证据 |
|---|---|---|
| 「素材」提取标记（#3 定稿降为论断字段，不另开节） | 提取当下顺手标一笔近乎零成本；事后从几百条论断里人肉挑素材成本极高 | 地图决策行、#3 定稿 |
| 逐字稿（含时间戳）永久资产 | 素材要用时得回到原话原语气——转述会丢掉神韵 | 地图决策行、#11 |

**MVP 不做**：不建素材检索界面、不做主题聚类。MVP 只保证素材标记在论断上。

## 愿景四：信源信用档案

**是什么**：给信源建立长期信用画像。节目转述外部信源（「据某报报道」「某记者披露」）。信源信用档案长期记录每个媒体、每位记者被引用了什么、后来证实还是证伪，逐渐形成「这家媒体/这位记者的历史准确率」——下次再看到该信源，就有历史成绩可查。

**依赖基座的廉价保险**：

| 基座字段 | 为什么信源信用档案需要它 | 证据 |
|---|---|---|
| 归因三层记录：链接 + 媒体 + **作者/记者名** | 记者名是三层里最容易被省掉的，但信用档案的最小单位恰恰是「人」；地图决策行明写「为将来的记者信用档案攒原料」 | 地图决策行 |
| `_index/sources.json` 归因补全结果，置信度与依据必填 | 补全的归因要能区分「节目明说的」和「程序猜的」，否则信用统计被猜测污染 | #3 定稿 |
| 断言/预测的核查结论（与求真引擎共享） | 「引用被证实/证伪」正是信用分的原料 | 推断（两愿景共享管线） |

**MVP 不做**：不算信用分、不建信源画像页。MVP 只保证归因记到记者名、补全结果带置信度落档。

## 廉价保险 ↔ 愿景对照总表

地图 Destination 提到建图会话定了「六条廉价保险」，但原始清单未留档。以下是依据地图决策行与 spec 重建的清单（同样以 vision issue 修正为准）：

| 廉价保险 | 消费者 | 落档处 |
|---|---|---|
| 「数据点」提取桶（含取数地址） | 资产分析框架 | 地图决策行、#3 |
| 「素材」提取标记 | 故事素材库 | 地图决策行、#3 |
| 预测类「期限/检验条件」字段 | 求真引擎 | 地图决策行、#3 |
| 归因三层（链接+媒体+记者名） | 信源信用档案 | 地图决策行 |
| 逐字稿（含时间戳）永久资产 | 全部四愿景的最终回链地 | 地图决策行、#11 |
| `_pairs` 校对配对（含 `engine:` 来源标记） | 不属于四愿景——服务回归测试与质量度量（训练用途已按最严口径关闭，#14） | spec §4 阶段 3、#14 |
